# Priority Cards

Priority Cards 用来决定先做什么、后做什么、暂缓什么。

## 高价值低风险先做

- 规则：收益明确、影响小、容易验证的任务优先
- 输出影响：标记为 P0

## 高风险高价值先会审

- 规则：价值高但可能影响架构、数据、发布或安全时，先会审
- 输出影响：推荐 multi-review 或 Claude Code 分析

## 低价值高成本暂缓

- 规则：收益不明确、成本高、验证困难时暂缓
- 输出影响：标记为 P2 或 backlog

## 可验证优先

- 规则：能快速证明成功或失败的任务先做
- 输出影响：验收标准必须具体

## 可回滚优先

- 规则：优先选择容易撤销的实现路径
- 输出影响：加入回滚说明

## 依赖少优先

- 规则：不新增服务、不新增库、不改配置的路径优先
- 输出影响：推荐最小修改

## 用户约束优先

- 规则：用户明确限制高于模型建议和速度
- 输出影响：约束写入 forbiddenChanges

## 阻塞项优先

- 规则：影响后续任务推进的 bug 或缺失信息优先处理
- 输出影响：标记为 P0

## 发布风险优先

- 规则：发布前先处理安全、隐私、构建、文档和回滚风险
- 输出影响：生成 release checklist

## 认知负担低优先

- 规则：在同等价值下，选择用户更容易理解和验收的路径
- 输出影响：任务包更短、更明确
